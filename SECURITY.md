## Reporting a Vulnerability

We take the security of our software seriously. If you believe you have found a security vulnerability, please report it to us as described below.

**Please do not report security vulnerabilities through public GitHub issues.**

### How to Report

Please report security vulnerabilities by emailing **<CONTACT_EMAIL>**.

In your report, please include:

1. A clear description of the vulnerability.
2. Steps to reproduce the issue (including code snippets or a repository if possible).
3. The version(s) of the project affected.
4. The potential impact of the vulnerability.

### Disclosure Policy

- We will work with you to understand and resolve the issue.
- We will fix the issue and release a patch as soon as possible.
- Once the issue is resolved, we will release a patch and you will be acknowledged in the release notes (unless you wish to remain anonymous).
- We ask that you refrain from publicly disclosing the vulnerability until we have had a chance to fix it.
